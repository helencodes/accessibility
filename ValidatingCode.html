Validating Your Site

Why Validate

  • Browsers are the helicopter parents of programming. Browsers are programmed to fix your errors but not all browsers fix code in the same way and not everyone is viewing the page through traditional browsers
  • As your page becomes more complex "hidden" areas can be devastating

3 Approaches to Validating

  1. Validate by URI/URL
    • won't work if you haven't paid for a domain yet
  2. Validate by filename
    • Instead of giving URL you can actually upload the file to check it
  3. Validate by direct input

Validating site to use is https://validator.w3.org

The W3 is the association that makes everyone stick to the same standards.

The validator may throw up lots of errors but Errors Propogate so usually if you fix the first one it solves multiple. Search online and in forums to ask about errors. The w3 validator will find most of the errors that break the rule of the DOM structure however it will occasionally miss some

Validate/Check The Accessibility

You can use Web Aim to validate your accessibility. (Https://wave.webaim.org) You can only do this with a URL (hosted pages)

Review
